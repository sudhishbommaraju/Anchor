Works with what you already use.

Anchor speaks the OpenAI and Anthropic wire formats and ships an MCP server. Point your tool at it with a base-URL + key swap — or read the key directly over REST.

ANTHROPIC_BASE_URL

Anthropic SDK

/v1/messages

MCP server

Expose get_context, report_step, and check as tools to any MCP client.

Direct REST

/v1/context

Can't swap the base URL? Read the key directly and treat injection_block as system context.

The only change you make.

Swap the base URL and key. That's it — everything else about your agent stays exactly the same.

# Point any OpenAI-compatible tool at Anchor
export OPENAI_BASE_URL="https://<appkey>.functions.insforge.app/anchor/v1"
export OPENAI_API_KEY="anc_live_…"

# …then use the SDK exactly as normal — streaming & tools pass through
import OpenAI from 'openai';
const client = new OpenAI({ baseURL: process.env.OPENAI_BASE_URL, apiKey: process.env.OPENAI_API_KEY });
# Claude Code / Anthropic SDK
export ANTHROPIC_BASE_URL="https://<appkey>.functions.insforge.app/anchor"
export ANTHROPIC_AUTH_TOKEN="anc_live_…"

# Anchor speaks /v1/messages — text + tools, streaming & non-streaming
# Any agent that can't swap its base URL: read the key directly
curl https://<appkey>.functions.insforge.app/anchor/v1/context \
  -H "authorization: Bearer anc_live_…"

# → { mission, current_step, memory, warnings, injection_block, guidance }
# As an MCP server for Claude Code
anchor_get_context   # full mission + memory + warnings
anchor_report_step   # report an action → updated context
anchor_check         # pre-flight a step for loop/drift
